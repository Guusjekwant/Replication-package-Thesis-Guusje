# Interview Coding and Analysis Dataset

## Overview

This repository contains the coding and analysis dataset used for the qualitative phase of the thesis research on the adoption of AI agents in organizations.

The dataset consists of interview excerpts, manually assigned codes, grouped themes, and intermediate analysis tables used to identify key factors influencing AI agent adoption. The coding process followed an inductive thematic analysis approach, in which interview fragments were coded, grouped into broader categories, and subsequently synthesized into findings.

**File included:**

* `Results.xlsx`

**Format:** Microsoft Excel (.xlsx)

**Purpose:** To provide transparency and traceability for the qualitative analysis and the development of the study findings.

---

# Coding and Analysis Process

The analysis was conducted in four main steps:

1. Interview transcripts were manually coded.
2. Relevant codes were extracted into spreadsheets.
3. Similar or overlapping codes were grouped into higher-level categories.
4. The grouped categories were analyzed and synthesized into the findings presented in the thesis.

The worksheets in this file represent different stages of this process.

---

# Worksheet Descriptions

## `All motivations`

### Description

Contains all interview excerpts that were coded as motivations for the use or adoption of AI agents.

### Contents

* Initial motivation code
* Supporting interview quotation
* Interview source

### Purpose

Serves as the raw coding dataset for the motivation analysis.

---

## `Grouped motivations`

### Description

Contains the grouped version of the motivation codes.

### Contents

* Higher-level motivation category
* Original coded motivation
* Supporting quotation
* Interview source

### Purpose

Used to identify recurring motivation themes across organizations.

---

## `Current phase long`

### Description

Contains coded excerpts from the long-form interviews regarding the organization's current stage in its AI journey.

### Contents

* Assigned adoption phase
* Supporting quotation
* Interview source

### Purpose

Used to classify organizations into AI adoption phases.

---

## `Current phase short`

### Description

Contains coded excerpts from the shorter conference interviews regarding AI adoption maturity.

### Contents

* Assigned adoption phase
* Supporting quotation
* Interview source

### Purpose

Provides additional validation and comparison of AI adoption stages.

---

## `Motivations long interviews`

### Description

Summarizes the motivation-related findings from the long interviews.

### Purpose

Used for frequency analysis and reporting of key motivations.

---

## `Motivations short interviews`

### Description

Summarizes motivation-related findings from the shorter interviews.

### Purpose

Used to compare patterns observed in the long interviews.

---

## `Use cases long`

### Description

Contains coded examples of AI agent use cases identified in the long interviews.

### Contents

* Use case category
* Supporting quotation
* Interview source

### Purpose

Used to identify common AI agent application areas.

---

## `Use cases short`

### Description

Contains AI use cases identified in the shorter interviews.

### Purpose

Used as supplementary evidence for use case analysis.

---

## `Factor Prioritization`

### Description

Contains coded statements regarding which factors respondents considered most and least influential for AI agent adoption.

### Purpose

Used to compare the relative importance of adoption factors across organizations.

---

## `Environmental factors`

### Description

Contains all codes classified under the environmental dimension of the Technology–Organization–Environment (TOE) framework.

### Examples

* Competitive pressure
* Compliance
* Costs
* Economic uncertainty
* Technology gap

### Purpose

Used to analyze external influences on AI agent adoption.

---

## `Organisational factors`

### Description

Contains all codes classified under the organizational dimension of the TOE framework.

### Examples

* Organizational readiness
* Change management
* Strategic alignment
* Organizational structure
* Innovation culture

### Purpose

Used to analyze internal organizational influences on AI agent adoption.

---

## Traceability

Each coded fragment can be traced back to:

* The original interview
* The assigned code
* The grouped category used in the analysis

This structure enables transparency and reproducibility of the qualitative coding process and the resulting findings.
